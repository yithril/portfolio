Project Summary

Full-stack education platform designed to teach real-world software development. Includes a custom-built LMS, structured course content, and AI-assisted learning features for feedback, grading, and content generation.

Problem

Learning to code is often:

unstructured
slow to get feedback
dependent on instructors for grading and support

Creating and maintaining high-quality course content is also time-intensive.

Approach
Built a custom LMS to structure courses, lessons, and learning paths
Integrated AI as a teaching assistant:
answers questions about course content
provides real-time feedback
evaluates student responses
Implemented AI-assisted grading:
reviews projects
gives feedback and scoring
Used AI to generate:
presentations
handouts
code examples
course materials

Focus was on combining structured learning + AI assistance, not just chat.

Tech Stack
Next.js
Supabase (Postgres, Auth, RPC)
TypeScript
AI APIs (LLMs, prompt workflows)
Challenges / Tradeoffs
Balancing AI flexibility with structured curriculum requirements
Ensuring AI feedback is useful and not generic
Avoiding over-reliance on AI vs real learning
Managing content generation quality
Outcome / Lessons
AI works best as a teaching assistant, not a replacement for structure
Strong curriculum design still matters more than AI alone
Combining LMS + AI creates a much better learning experience than either alone

🔥 Workflow IQ
Project Name

Workflow IQ

Slug: workflow-iq
Category: AI System / Agent Platform
Status: Prototype (Buildathon — 2nd Place)

Card Summary

Agent-based workflow system using DAG architecture to safely constrain and orchestrate AI agents.

Project Summary

System designed to make AI agents useful in real workflows while maintaining control, safety, and predictable behavior through structured inputs, outputs, and execution paths.

Problem

AI agents are powerful but risky:

too much access → unsafe
too little structure → unreliable
difficult to integrate into real systems
Approach
Built a DAG-based workflow system:
nodes represent steps in execution
conditional paths control flow
Agents are:
constrained
structured
limited in scope
Defined:
strict input/output contracts
allowed actions per agent
Goal:
make agents usable in real applications
without giving them full system access
Tech Stack
Next.js
Supabase
TypeScript
AI APIs
Challenges / Tradeoffs
Designing constraints without limiting usefulness
Balancing flexibility vs safety
Defining clear boundaries for agent behavior
Outcome / Lessons
Unconstrained agents are not production-ready
Structure and contracts are essential
DAG-based orchestration is a strong model for AI workflows

Drive-Thru AI
Project Name

Drive-Thru AI

Slug: drive-thru-ai
Category: Applied AI System / Conversational AI
Status: Completed (Capstone)

Card Summary

Conversational AI system simulating a fully automated drive-thru ordering experience.

Project Summary

End-to-end conversational AI system where the model acts as a drive-thru order taker, handling real-world, messy user input and maintaining context across a full ordering session.

Problem

Real-world conversations are:

noisy
non-linear
full of interruptions and corrections

Most AI demos assume:

clean input
linear conversations
ideal user behavior

This doesn’t reflect real usage.

Approach
Designed a conversational system that:
maintains context across multiple turns
handles interruptions and order changes
filters out irrelevant or extraneous conversation
Simulated real-world interaction patterns:
users changing orders mid-sentence
adding/removing items dynamically
Focused on:
robustness
adaptability
realistic conversational flow
Tech Stack
Next.js
Supabase
TypeScript
AI APIs (LLMs, prompt orchestration)
Challenges / Tradeoffs
Managing conversational state across multiple turns
Preventing the AI from drifting or losing context
Handling ambiguous or incomplete input
Designing prompts that are flexible but controlled
Outcome / Lessons
Real-world conversational systems require state management, not just prompts
AI must handle messy input, not idealized input
Robustness matters more than clever responses

AI-powered legal analysis system built to scale to millions of documents using Elasticsearch.

Project Summary

System designed for legal discovery workflows, combining large-scale document indexing with AI-driven summarization, categorization, and structured information extraction.

Problem

Legal discovery involves:

massive volumes of documents
unstructured text
difficulty identifying key relationships and timelines

Traditional approaches are:

time-consuming
manual
difficult to scale
Approach
Used Elasticsearch to:
index and query large document sets
support scaling to millions of records
Applied AI to:
summarize documents
categorize content
extract key entities
Built structured outputs:
timeline of events from document content
person/entity relationship map
Focused on turning unstructured data into usable, structured insights
Tech Stack
Next.js
Supabase
Elasticsearch
TypeScript
AI APIs (LLMs, embeddings)
Challenges / Tradeoffs
Scaling search and retrieval across large datasets
Ensuring AI summaries are accurate and useful
Extracting structured data from inconsistent text
Balancing performance vs depth of analysis
Outcome / Lessons
AI is most valuable when paired with strong search infrastructure
Structured outputs (timelines, relationships) are more useful than raw summaries
Combining search + AI enables scalable analysis systems

Card Summary

AI-powered travel recommendation system using RAG and web-scraped data to surface lesser-known destinations in Michigan.

Project Summary

Hackathon project designed to promote tourism to lesser-known locations by aggregating and analyzing data from sources like Yelp and Reddit, then using AI to generate personalized travel recommendations.

Problem

focus on popular destinations
surface the same recommendations repeatedly
ignore local or lesser-known places

tourism diversity
discovery of hidden gems
Approach
Collected data from multiple sources:
Yelp

public web content
Built a RAG (Retrieval-Augmented Generation) pipeline:
indexed scraped data
retrieved relevant context based on queries
Used AI to:
generate recommendations
summarize locations
provide contextual suggestions
Focused on surfacing:
unique
lesser-known
locally relevant destinations
Tech Stack
Next.js
Supabase
Web scraping pipelines
RAG architecture
Vector embeddings
AI APIs
Challenges / Tradeoffs
Cleaning and normalizing inconsistent scraped data
Ensuring relevance and quality of recommendations
Avoiding generic or repetitive AI responses
Time constraints of hackathon development
Outcome / Lessons
RAG is effective for grounding AI in real-world data
Data quality has a major impact on output quality
Even simple pipelines can produce useful discovery tools
Hackathons are great for rapidly testing applied AI ideas
